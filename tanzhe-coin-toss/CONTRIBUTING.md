# 贡献指南

感谢你愿意帮助改进这个项目。我们欢迎 Bug 修复、玩法建议、视觉与声音优化、无障碍改进和文档完善。

## 开始之前

1. 先搜索现有 Issue，避免重复讨论。
2. 较大的玩法或结构调整，请先创建 Feature Request 说明动机与方案。
3. 不要提交未经授权的照片、音乐、字体、商标或其他第三方素材。

## 本地开发

项目没有依赖和构建步骤。修改 `index.html` 后直接用现代浏览器打开即可。

`index.html` 内部依次包含：

- HTML 页面结构
- 内联 CSS 场景与响应式样式
- 内联 JavaScript 游戏状态、碰撞判定和 Web Audio 声音

更详细的实现说明见 [docs/architecture.md](docs/architecture.md)。

## 提交前检查

- 桌面端鼠标能在整个游戏区域移动准星。
- 按住空格开始蓄力，松开后只投出一枚铜钱。
- 触屏两次点击模式可完成蓄力和投掷。
- 铃铛、铜钱和落空都具有可区分的声音与视觉反馈。
- 关闭“乐♪”后，BGM 和所有效果音均停止。
- 390px 宽度下没有横向滚动或被遮挡的主要控件。
- 开启“减少动态效果”系统设置后，游戏仍可操作。
- 页面没有网络请求、外部脚本或用户追踪。

## Pull Request

- 一个 PR 尽量只解决一个问题。
- 标题用动词开头，例如“改善移动端投掷按钮反馈”。
- 描述修改原因、实现方式和手动测试结果。
- 涉及视觉变化时，欢迎附上修改前后的截图。

提交贡献即表示你同意自己的贡献按照本项目的 MIT License 发布。
