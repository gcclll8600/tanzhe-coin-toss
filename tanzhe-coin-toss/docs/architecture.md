# 实现说明

## 设计目标

项目采用单文件架构，目标是让任何人下载后双击即可试玩，也方便在 GitHub Pages、对象存储或普通静态服务器上部署。

## 页面结构

`index.html` 包含三层：

1. **场景层**：福字墙、支架、铜钱、铃铛、地面和准星。
2. **交互层**：蓄力条、投掷按钮、结果提示和计分信息。
3. **声音层**：通过 Web Audio API 实时合成 BGM、铃声、铜声和落空声。

视觉元素全部由 HTML 与 CSS 构成，不依赖图片或远程字体。README 封面不参与游戏运行。

## 游戏循环

1. 指针移动或触屏点击更新归一化准星坐标。
2. 空格键或按钮启动循环蓄力。
3. 投掷时根据力度与投掷次数计算轻微偏移。
4. 将最终落点与每枚铜钱中心比较：
   - 距离小于铃铛半径：`bell`
   - 距离小于铜钱主体半径：`coin`
   - 其他情况：`miss`
5. 根据结果播放动画、声音并更新分数。

## 声音系统

- BGM 使用五声音阶的短音符序列循环合成。
- 铃声由多个正弦与三角振荡器叠加。
- 铜钱撞击使用快速下降的低频三角波。
- 落空由带通噪声和低频木质敲击组合。

声音只会在用户首次交互后启动，以符合浏览器自动播放政策。

## 数据与隐私

项目没有服务器、Cookie、分析脚本或本地存储。所有游戏状态都只存在于当前页面内存中，刷新页面后重置。
